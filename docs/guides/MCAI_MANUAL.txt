============================================================
2. SAFETY RULES
============================================================

Owner:

   Only ModVinny should command tj.

Emergency stop:

   tj stop
   tj cancel
   tj halt

These should cancel movement, active tasks, active skills, curriculum steps, gear tasks, villager tasks, blueprint builds, and bridge-triggered operations.

LLM safety:

   Ollama may help with dialogue and explanation.
   Ollama must not verify evidence.
   Ollama must not execute raw actions.
   Ollama must not generate JavaScript for tj to run.
   No OpenAI API is used.
   No cloud API is used.

Risky operations:

   Risky operations require confirmation.
   Examples: Nether entry, portal lighting, villager trades, enchanting, anvil use, potion use, blueprint building, bridge region changes.

Honest unsupported features:

   Brewing is scaffolded unless reliable API support exists.
   Schematic import is disabled by default.
   The server plugin bridge is optional.

4. NATURAL SPEECH COMMANDS
============================================================

You do not have to memorize every exact command. tj now has a natural command router that translates common owner phrases into existing safe registered commands.

Examples:

   tj we need food
      Routes to: tj get food

   tj get wood
      Routes to: tj gather wood

   tj find coal
      Routes to: tj mine coal

   tj get 16 coal
      Starts a coal run for 16 coal instead of stopping after one block.

   tj mine 12 stone
      Mines up to 12 safe stone/cobblestone blocks.

   tj collect 6 logs
      Starts a wood run for 6 logs.

   tj lets get ready for mining
      Routes to: tj prepare for mining

   tj make camp
      Routes to: tj make camp

   tj make storage
      Routes to: tj place storage chest

   tj light this place
      Routes to: tj light home

   tj what should we do next
      Routes to a progression/advisor command.

   tj get better gear
      Routes to: tj suggest gear upgrades

   tj check if we are ready for the Nether
      Routes to: tj nether checklist

   tj find good trades
      Routes to: tj best trades

   tj build a small shelter
      Routes to the small-shelter blueprint build flow.

Ambiguous phrases ask a question instead of guessing:

   tj make us safe
      tj may ask whether you mean light home, build shelter, or protect you.

Risky phrases still require the existing confirmation flow:

   tj go nether
      tj should not enter automatically. It should require Nether confirmation.

Unsupported phrases are refused honestly:

   tj build a giant castle
      tj should explain that huge builds are not supported yet and offer small blueprints instead.

Debug natural routing:

   tj natural router status
   tj what did you think i meant
   tj clear pending intent
   tj natural examples
   tj natural test: we need food

Important safety notes:

   Emergency stop still wins first: tj stop, tj cancel, tj halt.
   Exact commands still run before natural routing.
   Only ModVinny can trigger natural action commands.
   Ollama may help classify unclear phrases, but only from existing registered commands.
   Ollama cannot invent commands, execute raw actions, or bypass safety.

4.5. COMPETENCY, CORRECTIONS, AND LEARNING
============================================================

tj can now learn owner-approved natural phrases after clarification.

Example:

   ModVinny: tj make us safer
   tj: I can do that a few ways: light home, build shelter, or protect you. Which one?
   ModVinny: light home
   tj: runs light home and remembers that "make us safer" means "tj light home".

Rules:

   Only ModVinny can teach mappings.
   Learned mappings must point to existing registered commands.
   Risky learned mappings still require the normal confirmation flow.
   Learned mappings do not create new actions.
   Ollama still cannot execute raw actions.

Competency commands:

   tj competency
      Reports reliable, improving, shaky, blocked, and untested skill counts.

   tj what are you good at?
      Same as competency.

   tj reliable skills
      Lists skills with enough successful evidence.

   tj shaky skills
      Lists skills with weak or failing evidence.

   tj untested skills
      Lists skills that need more real tests.

Command learning commands:

   tj learned commands
   tj command learning
   tj natural learning
      Lists learned natural phrase mappings.

   tj learn commands on
   tj learn commands off
      Enables or disables natural command learning.

Mode commands:

   tj mode
      Shows current interaction mode.

   tj careful mode
      Asks before more actions and avoids auto-running low-risk natural commands.

   tj helper mode
      Default mode. Auto-runs clear low-risk natural commands.

   tj quiet mode
      Keeps chat shorter.

   tj explain mode
      Gives short reasons and failure explanations.

   tj test mode
      Keeps extra logging and test-friendly explanations on.

Failure recovery:

   If tj fails, he should explain the practical blocker and suggest safe existing commands.

   Example:
      "I can't craft torches because I don't have coal. I can mine coal or gather wood."

Manual test commands:

   tj test plan
   tj test natural commands
   tj test core helper
   tj test survival basics
   tj test report

Session recorder:

   tj records high-level route decisions, failures, recovery suggestions, and stop events in session-log.jsonl when enabled.
   It redacts tokens, secrets, and local paths.
